pi-cyber-ui

Design Language · Tokyo Night cyber 派生 · 终端 TUI 沉淀

克制、低饱和、有层级。每个颜色绑定一个语义角色；同一区域不出现两个相同色级；动效低频不抢戏。

1 · 设计原则

  1. 语义优先。颜色不是装饰，是状态/角色标签。cyan 仅留给主品牌 / 当前焦点 / 进度主调。
  2. 层级唯一。同一行/区域不重复同色级。读 → 看 → 操作三层视觉权重必须有明显落差。
  3. 克制动效。动效服务于"还在跑"的信号，不喧宾夺主。呼吸/letter-wave 周期 ≥ 1.8s，不闪。
  4. 可降级。宽度不足时从右向左丢段；颜色失败回落到 fg/fgDim；不依赖 24-bit color 的形状/语义。
  5. 设计先行。UI 改动先写 HTML mock，review 通过再落 code。

3.4 Tool rendering

覆盖 read / bash / edit / write / grep / find / ls 的 UI render；执行层继续委托 Pi built-in tools。renderShell: "default"；保留内置 parameters/description/promptSnippet/promptGuidelines（...initial.<tool> spread）。

段规则
pathbasename fg+bold；~ hotPink；shortenPathToWidth 自适应。

thinkingoff=thinkingOff · minimal=tealDark · low=blue · medium=cyan · high=purple · xhigh=pink+bold；high 及以上彩虹。

6 · 不变量

① TPS 算法（cumulative）和 token 计算逻辑 — 不动。
② 注册顺序：editor(pi) 必须在 working(pi) 之前（producer → consumer）。
③ 工具内置 parameters / description / promptSnippet / promptGuidelines 必须 spread 保留；execute 必须委托 Pi built-in。
④ 工具 render 是本包定制层；Pi native renderer 新特性必须 review/port，不能假设自动继承。
⑤ read 必须保留 Pi compact 语义：SKILL.md → [skill] name；AGENTS.md/CLAUDE.md → resource。
⑥ 不在 working line 重复展示工具运行（HUD 已有）。
⑦ cyan 不下放到普通文本/路径/工具状态符。
⑧ silver 系仅限 working line 内部使用。
⑨ 不引入二进制串 / emoji 串 / CSS gradient 类视觉噱头。

7 · 流程

  1. UI 改动 → 先写/改 design HTML mock。
  2. 用户 review HTML → 确认。
  3. 落 code，npm run typecheck。
  4. 版本：npm version patch --no-git-tag-version。
  5. 不创建 git tag；不要 npm pack。
