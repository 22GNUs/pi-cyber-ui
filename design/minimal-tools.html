5 实现改动锚点

模块改动说明
cyber-editor.ts 大改 移除 renderHudChrome 调用 + 顶部/底部 rail 拼接逻辑；保留 ❯ 与 border。
editor-hud.ts 删除/迁移 核心拼装逻辑（path / turn / token / tps）拆出去：path 给 footer，turn/token/tps 给 working。
working.ts 大改 双行 message：第一行 spinner+verb+timer+esc；第二行 turn + tool tally + ↑↓ + tps。
新增 idle summary：agent_end 后保留 last summary 直到下次 agent_start。
footer.ts 扩展 当前显示 model / ctx / thinking。新增：
• cwd（来自 ctx.cwd，复用现有 shortenPath）
• git 分支 + 脏文件计数（来自 FooterDataProvider，pi 已暴露 branch）
editor-state.ts 不变 snapshot 仍生产 turn/token/tps/toolStats，由 working 消费而非 HUD。
tool-render.ts 不变 tool 行渲染独立于 HUD 重组。

6 行为细节

  • working 第一行：<spinner> <verb> · <elapsed>，elapsed ≥ 10s 追加 esc to cancel。
  • working 第二行：t<n> · <tool spinner+name> · ↑in ↓out · tps。无 tool 跑时省略 tool 段。
  • idle summary：agent_end 后切换为 ✓ done · <total elapsed> + 第二行 t<total> · ✓N ✗M · ↑in ↓out · avg tps，左侧 border 切 green。
  • 下一次 prompt：agent_start 立即覆盖 last summary 进入运行态。
  • 窄屏：单行宽度不足时优先丢弃 tps → ↑↓ → tool name，保留 spinner+verb+timer。
  • footer git：异步取 branch + 脏文件数；脏数为 0 时不显示 ✚n。
